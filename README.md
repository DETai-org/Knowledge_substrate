# DET / DETai — Knowledge Core

Внутренняя каноническая база знаний экосистемы **DET / DETai**. Репозиторий хранит исходные Markdown-материалы и публикует их через **MkDocs Material** на **GitHub Pages**.

## Что внутри

- **docs/** — структура знаний (RU + EN), отражающая экосистему:
  - **Onboarding** — маршрут входа и правила чтения
  - **DET** — смысловой слой: язык, принципы, миссия
  - **DETai** — экосистема, проекты и предпроекты
  - **U.L.I. (Улей)** — стандарты, процессы, инфраструктура
- **mkdocs.yml** — навигация и конфигурация сайта
- **.github/workflows/docs.yml** — автосборка и деплой

## Быстрый старт (локально)

> Требуется Python 3.11+

```bash
pip install mkdocs-material
mkdocs serve
```

После запуска документация будет доступна по адресу `http://127.0.0.1:8000/`.

## Деплой

Деплой выполняется автоматически при пуше в ветку `main`, если изменялись:
- `docs/**`
- `mkdocs.yml`
- `.github/workflows/docs.yml`

## Принципы ведения

- **Markdown — канон знаний**: изменения обсуждаются и фиксируются в PR.
- **Структура важнее наполнения**: каркас должен быть стабилен.
- **RU + EN**: минимальные англоязычные страницы поддерживаются для навигации.

## Как добавлять материалы

1. Добавьте или обновите Markdown-файл в `docs/`.
2. При необходимости обновите `mkdocs.yml` (навигация).
3. Откройте PR с кратким описанием изменений.

---

Если нужно расширить структуру, сначала согласуйте изменения в навигации.
