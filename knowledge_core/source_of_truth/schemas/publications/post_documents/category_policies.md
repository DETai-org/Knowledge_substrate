# Category Policies

**v1.0** ♻️

## Назначение документа

Этот документ описывает правила работы с **categories** в документах типа **post**
и фиксирует их роль в таксономии публикаций.

---

## 1. Роль categories

`category_ids` — это **поперечные фильтры**, уточняющие аспект рассмотрения темы.
Категория отвечает на вопрос: **«С какой стороны рассматривается тема?»**

Categories:

- не определяют смысл документа (это роль rubric);
- уточняют ракурс, масштаб или контекст;
- помогают навигации и аналитике.

---

## 2. Правила использования

- Для `type = post` требуется **ровно одна** category.
- Для остальных типов документов categories **не обязательны**.
- В `taxonomy` используются только **ID** из `categories.json`.

---

## 3. Связь с controlled vocabularies

Categories являются управляемым словарём и подчиняются общим правилам:
стабильность `id`, lifecycle (`candidate | approved | deprecated`),
запрет на физическое удаление.

---

## 4. Анти-паттерны

❌ Использовать category вместо rubric
❌ Хранить строки вместо ID
❌ Назначать категорию «для охвата»
