# Структура `docs/` — три сущности

Внутри `docs/` сосуществуют три разные сущности, каждая со своей логикой:

1. **Ecosystem** — база знаний об экосистеме DET / DETai.  
   Мультиязычная, организована через языковой сегмент в пути: `docs/{lang}/ecosystem/...`,  

2. **Publications** — канон публикаций (посты, статьи, исследования).  

3. **Assets** — технический код и скрипты

________

## ecosystem

Русская версия (RU) является **дефолтной** и живёт в `ru/ecosystem`.

Остальные языки — это **зеркала** структуры RU, размещённые в подпапках:

* `docs/en/…` — English
* `docs/fi/…` — Suomi
* `docs/de/…` — Deutsch

## Принцип “просто зеркалируем”

* Когда мы добавляем или обновляем страницу на RU, мы при необходимости создаём соответствующую страницу в нужном языке **с тем же относительным путём**.
* Пример:
  * RU: `docs/ru/ecosystem/DET/index.md`
  * EN: `docs/en/ecosystem/DET/index.md`
  * FI: `docs/fi/ecosystem/DET/index.md`
  * DE: `docs/de/ecosystem/DET/index.md`

Переводы можно добавлять постепенно:

* пока страницы нет — допускается заглушка “coming soon” + ссылка на RU-версию
* язык считается “готовым”, когда ключевые входные страницы (`index.md`, `ecosystem/index.md`) переведены

Такая схема позволяет масштабировать сайт на 2/4/10 языков без изменения архитектуры: мы просто **расширяем зеркала**.

## publications/
Домен публикаций хранится отдельно и **не переводится**:
- `docs/publications/...` — посты, статьи исследования

## Технические ассеты (не контент)
Внутри `docs/` также есть assets/` (внутри которой уже, `javascripts/`, `stylesheets/`и прочее ), 
Весь assets/ служат MkDocs (JS/CSS и т.п.) и **не являются знанием**.
