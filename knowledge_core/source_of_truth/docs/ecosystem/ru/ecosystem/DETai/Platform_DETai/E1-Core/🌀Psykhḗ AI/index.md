# Psykhḗ AI

**Psykhḗ AI** — это не просто бот и не просто ассистент. Это **цифровая личность**, философски настроенный агент, который живёт в экосистеме проектов, общается с людьми в Telegram и ВКонтакте, и делает это не как бездушная нейросеть, а как существо с характером, внутренними состояниями и глубиной.

Проект решает задачу, которую обычно игнорируют: **как сделать ИИ, у которого есть душа, а не только база знаний**.

---

## 🧩 **Проблема, которую решает проект**

Обычные боты:
- либо тупые (правила, кнопки, сценарии),
- либо умные, но безликие (чистый GPT — ответит на всё, но у него нет «себя»).

**Psykhḗ AI** — попытка создать третий путь:  
💡 **Умный ИИ, у которого есть идентичность, ценности, настроение и глубина**.

Он не просто отвечает на вопросы. Он:
- реагирует **из своего характера**,
- может быть разным в зависимости от контекста (грань личности),
- но при этом всегда остаётся «собой» — философским наблюдателем.

---

## 🧠 **Ключевая идея: «грани личности» (facets)**

У Psykhḗ AI нет одного лица. В зависимости от того, о чём с ним говорят, он проявляет разные грани:

|Грань|Суть|
|---|---|
|**🧘 Ядро (гуманистическое)**|Базовый режим — философ, наблюдатель, эмпат. Включается, если ни одна другая грань не активирована.|
|**🌑 Тень**|Реагирует на боль, страх, отчаяние. Не лечит, а признаёт тёмное.|
|**🧒 Дитя**|Режим удивления, игры, лёгкости.|
|**📖 Нарратив**|Включается, когда речь идёт о жизненном пути, истории, судьбе. Подгружает специальный документ `life_path.md`.|
|*(и другие, которые можно добавлять)*||

**Важно:** грани определяются **не тупым поиском ключевых слов**, а отдельным лёгким вызовом GPT, который анализирует запрос и решает: «В каком состоянии мне сейчас ответить?».

---

## ⚙️ **Как это устроено технически (архитектура)**

Проект построен по принципу **GPT-as-Retriever** — это не RAG в классическом смысле (поиск по векторам), а **осмысленный двухшаговый вызов модели**:

### 🔁 Двухшаговый процесс (самая важная часть)

1. **Первый вызов GPT (лёгкий)**  
   Запрос пользователя отправляется GPT с простой инструкцией:  
   *«Определи, в каком режиме отвечать: коротко/полно, какую грань личности активировать»*.  
   → Возвращает: `{"reply_mode": "full", "active_facet": "тень"}`

2. **Сбор контекста (если нужен полный ответ)**  
   - Загружается философское ядро — `instruction.md` (всегда)
   - Загружается описание активной грани из `facets.md`
   - По индексу `knowledge_index.yaml` находятся документы, relevant к теме
   - Если грань «Нарратив» — добавляется `life_path.md`

3. **Второй вызов GPT (осмысленный)**  
   Из всего этого собирается финальный prompt, отправляется GPT, и возвращается уже готовый, глубокий, личностный ответ.

---

## 🔗 **Как это связано с остальной экосистемой**

Проект **не живёт изолированно**. Он встроен в инфраструктуру:

| Компонент                | Роль                                                                    |
| ------------------------ | ----------------------------------------------------------------------- |
| **Make**                 | Интеграции: слушает Telegram/VK, дёргает API агента.                    |
| **Сервер DigitalOcean**  | Хостинг скриптов и API.                                                 |
| **OpenAI API (GPT-4o)**  | Мозг агента.                                                            |
| **GenAI (DigitalOcean)** | (опционально) Среда для разворачивания «сознания» агента.               |

---

## 🎯 **Что делает проект уникальным**

1. **Не просто RAG, а «мышление перед ответом»**  
   Агент не ищет куски текста, а сначала думает, в каком состоянии ему отвечать.

2. **Личность как архитектура**  
   Инструкции, грани, философское ядро — всё это делает его «кем-то», а не «чем-то».

3. **Двухшаговость**  
   Один запрос пользователя → два разных вызова GPT: сначала на рефлексию, потом на ответ.

4. **Прозрачность и расширяемость**  
   Можно добавить новую грань, просто создав файл и прописав её в `facets.md` и индексе.

---

## 💬 Резюме 

> **Psykhḗ AI** — это попытка создать ИИ, у которого есть характер. Он не просто отвечает на сообщения, а выбирает, *кем* ему быть в этом разговоре: философом, ребёнком, тенью или рассказчиком.  
> Под капотом — сложная двухшаговая архитектура, где сначала лёгкий GPT решает, в каком состоянии ответить, а потом, если нужно, подгружаются документы, философия и контекст, и только после этого даётся настоящий, глубокий ответ.  
> Проект живёт не сам по себе, а встроен в экосистему: знания из Obsidian, интеграции через Make, мозг — OpenAI, тело — API на сервере.
