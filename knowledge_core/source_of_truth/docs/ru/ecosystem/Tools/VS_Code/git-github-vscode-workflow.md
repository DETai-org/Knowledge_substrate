# Git, GitHub и VS Code: рабочая модель с участием агента (Codex)




> [!NOTE]
> Этот документ описывает рабочую модель в условиях,
> когда Codex / агент работает через **GitHub Web (Pull Requests)**.
> 
> С выходом **Codex App (Desktop)** базовая логика документа
> сохраняется, а локальный VS Code становится
> основным рабочим интерфейсом вместо веб-версии GitHub.

---


### Правильная best practice 

✅ **На сервере не правим.**  
Сервер = деплой/рантайм.  
Правки = только в репозитории (локально/агент/PR).

Это резко снижает хаос.

---
##  если ты и в Web (PR агента), и локально в VS Code?

Вот тут ключ: **можно работать и там, и там**, просто нужно одно правило, которое всё стабилизирует.

### Золотое правило

**Перед началом локальной работы всегда делай Pull (или Sync).**  
И после локальной работы — **Push**.

Это занимает 2 секунды, но убирает 90% конфликтов.

---

## 4) Текущий реальный режим: агент делает PR → ты мерджишь в web

Это вообще отлично. Но после мерджа в `main` происходит важное:

- **GitHub обновился**
    
- твой локальный клон **остался на старом состоянии**
    

И это нормально.

### Что делать?

Когда после мерджа садишься работать руками в VS Code:

1. **Pull** (забрать свежий `main`)
    
2. правки
    
3. commit
    
4. push
    

Всё. Никакой “синхронизации окружений”, просто актуализация копии.

---

## 5) “Не редактируй и там и там” ❓

Уточню: опасно не “и там и там вообще”, а опасно:

> **править одну и ту же ветку, не подтягивая изменения между правками**

Если ты:

- мерджишь PR в web,
    
- потом делаешь pull локально,
    
- работаешь локально,
    
- пушишь обратно,
    

— это абсолютно нормальный поток.

---

## 6) Практический безопасный протокол (простое правило на каждый день)

### Когда агент (Codex) сделал PR

- Ты смотришь PR в web
    
- Мерджишь в `main`
    

### Когда ты сам садишься править руками

1. Открыл VS Code
    
2. Нажал **Pull** (или `Sync`)
    
3. Поработал
    
4. **Commit**
    
5. **Push**
    

### Когда снова пошёл смотреть PR агента

- ок, смотри в web
    
- мерджишь
    
- перед следующей локальной работой опять **Pull**
    

🔁 Это цикл. Он простой и не ломается.
