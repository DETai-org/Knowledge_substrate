# VS Code


**Visual Studio Code** — это лёгкая, но мощная среда для работы с кодом, текстами и проектами.

VS Code — это:

- редактор кода,
    
- рабочее пространство,
    
- интерфейс для взаимодействия с файлами, серверами и репозиториями.
    

Он используется для:

- написания и редактирования кода,
    
- работы с Git-репозиториями,
    
- запуска команд через терминал,
    
- подключения к удалённым средам (серверы, Codespaces).
    

Важно понимать, что VS Code **сам по себе не является GitHub’ом и не является сервером**.  
Он — **универсальный интерфейс**, который позволяет работать с кодом и проектами в разных средах.

Именно поэтому VS Code можно рассматривать не просто как инструмент, а как **точку входа в разные уровни инфраструктуры**.

---

### Роль VS Code в экосистеме

В экосистеме DETai VS Code используется преимущественно как **интерфейс для операционной среды** — то есть для прямого подключения к серверу и работы с уже существующим кодом и конфигурациями.

Для разработки, экспериментов и проверки новых идей используется отдельная лабораторная среда (например, GitHub Codespaces или локальная разработка), чтобы не нагружать и не рисковать рабочими серверами.

Таким образом, VS Code участвует в системе не как место, где «живёт» проект, а как средство доступа к тем средам, где проект разрабатывается или эксплуатируется.

---

#### **Карта рабочих сред: лаборатория → GitHub → сервер**

В нашей системе есть два уровня, и важно не путать их роли:

- **Лаборатория (GitHub)** — место, где мы создаём и проверяем код без риска для рабочих серверов.  
    Примеры лаборатории: **GitHub и/или Codespaces**, локальная среда разработчика.
    
- **Сервер (операционная / production)** — место, куда попадает **уже проверенный код**.  
    На сервер не «экспериментируют», а разворачивают и поддерживают работающие версии.
    

👉 Принцип простой: **сначала проверяем в лаборатории → фиксируем в GitHub → только затем доставляем на сервер**.

_(См. также: «GitHub».)_

---

#### **Роль VS Code в этой схеме**

VS Code — это **интерфейс**, который может подключать тебя к разным уровням:

- **VS Code (локально)** — работа с файлами на компьютере.
    
- **VS Code + Codespaces** — работа в лаборатории (удалённая VM GitHub).
    
- **VS Code + Remote SSH** — вход в операционную зону (файлы и процессы на сервере).
