

# GitHub

**GitHub** — это главная мировая площадка, где живёт и развивается код. Если совсем просто: представьте Google Диск, только для программного кода, но с суперспособностями.

Принадлежит компании **Microsoft** (купили в 2018 году), и это хорошо — потому что GitHub теперь тесно связан со всеми современными инструментами разработки.

## Зачем это вообще нужно?

Когда вы пишете код (или промпты, или документацию — всё это тоже код), у вас на компьютере папка с файлами. GitHub — это **та же самая папка, но в интернете**, и с магией:

- **История как в Google Документах** — можно откатиться к любой версии файла, даже если неделю назад всё сломали
- **Работа в команде** — несколько человек могут менять одни и те же файлы, и GitHub сам разберётся, как склеить изменения
- **Ветки** — можно сделать копию всего проекта (ветку), поэкспериментировать, а потом влить обратно в основную версию, если эксперимент удался

## Что вайп-кодеру нужно знать про GitHub

GitHub — это **место, где встречаются код и люди**. Для вас как для новичка это значит:

**1. Там хранятся исходники всего**
Любой открытый проект, любая библиотека, любой инструмент — скорее всего, лежит на GitHub. Хотите посмотреть, как устроен популярный сайт или бот? Идёте на GitHub.

**2. Это соцсеть для кода**
У каждого проекта есть Issues — что-то вроде «задач» или «обсуждений». Там люди сообщают об ошибках, предлагают идеи, спорят о решениях. Pull Requests — это предложения изменений: кто-то что-то поправил и предлагает добавить это в проект.

**3. Там есть автоматизация**
GitHub Actions — это когда при каждом вашем действии (например, вы отправили файл на GitHub) может автоматически что-то происходить: запуститься проверка, собраться сайт, отправиться уведомление в Telegram. Всё настраивается без сложных серверов.

## А GitHub — это то же самое, что Git?

Нет, и это важно:

- **Git** — это система, которая живёт у вас на компьютере и просто отслеживает изменения в файлах (версии, ветки, историю)
- **GitHub** — это облачный сервис, где эти версии можно хранить, показывать другим и обсуждать

Git — как двигатель машины. GitHub — как автосалон, где эти машины выставляют, продают, обсуждают и чинят коллективно.

## Почему Microsoft купила GitHub?

Потому что GitHub — это **база всей современной разработки**. Без него невозможно представить ни open-source, ни корпоративную разработку. Microsoft сделала GitHub своим «фасадом» для разработчиков: теперь он идеально дружит с их редактором кода VS Code и остальными инструментами.
