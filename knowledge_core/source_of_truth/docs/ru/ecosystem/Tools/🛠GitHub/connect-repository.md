
# Подключить репозиторий
✅



Инструкция: как подключать новые репозитории к автоматической синхронизации между  GitHub и Сервером. ➕ См. в [uli_core](https://github.com/DETai-org/projects/tree/main/uli_core#readme)
____

> Перед началом убедись, что на сервере уже есть SSH-ключ (`~/.ssh/id_ed25519.pub`) и что он добавлен в GitHub (Settings → SSH and GPG keys).

Проверить подключение:
```bash
ssh -T git@github.com
```
> Если видишь `Hi Anton-Psy! You've successfully authenticated...` — всё готово.
> Обычно это надо только когда прикрепляем репозиторий к **новому серверу** 🆕 если к серверу уже привязан ключ что этот процесс Уже не надо делать

> [!WARNING]
> Если в локальной директории уже есть свои файлы (например, README.md), то при создании репозитория на GitHub лучше **не добавлять README.md и .gitignore**.  
> Иначе придётся вручную сливать конфликтующие файлы.

## 🔁 Шаги для подключения нового репозитория к автосинхронизации


### 🧩 1. Создай новый репозиторий

На GitHub:

- Назови его, например, `site`
    
-  **Луше не добавь** `.gitignore` (Python) 
- **Луше не добавь**  `README.md`
    
- Сделай **private**, если не хочешь открытый доступ.
    

---

### 🧲 2. Клонируй с сервер 

🧩 **Когда использовать:**  
Ты уже создал репозиторий на GitHub (например, `Wordpress`), и хочешь скопировать содержимое соответствующей папки с сервера туда.

> [!NOTE] по умолчанию создаётся `master`, а GitHub использует `main`. Чтобы избежать ошибок:


> После `git init` сразу переименуй ветку:
```bash
 git branch -M main
```
> Это избавит от несостыковок при push.


На сервере (Yandex Cloud):

```bash
# 1. Перейди в нужную папку на сервере
cd root/ИМЯ_ПОЛЬЗОВАТЕЛЯ/ИМЯ_ПАПКИ

# 2. Инициализируй Git
git init

# 3. Добавь GitHub как origin
git remote add origin git@github.com:DETai-org
/ИМЯ_РЕПОЗИТОРИЯ.git

# 4. Скачай содержимое GitHub (если оно есть)
git pull origin main --allow-unrelated-histories

# 5. Добавь все локальные файлы
git add .

# 6. Сделай коммит
git commit -m "Объединение с GitHub-репозиторием"

# 7. Запушь всё в репозиторий
git push -u origin main
```

📌 После этого:
- Содержимое папки окажется на GitHub

___

### 🧾 4. Добавь папку autosync/ 

В неё нужно поместить два файла:

- **auto_sync_to_github.sh** — скрипт, который следит за изменениями и пушит их на GitHub.
- **auto-sync-github.service** — сервис для systemd, чтобы скрипт автоматически запускался и работал в фоне.
    
👉 Эти файлы уже есть в репозитории `psy-ru`. Достаточно скопировать и поправить путь к проекту.

### 🧾 Добавь `requirements.txt`

📌 В корне репозитория (рядом с `.github/` и `autosync/`) создай файл:

```
requirements.txt
```

Туда будут собираться зависимости Python-проектов.  
Это нужно для **AI-Агента Codex**.
➕ 
Удобно, чтобы при деплое на сервер сразу ставить пакеты командой:

```bash
pip install -r requirements.txt
```


---

### 🟢 Проверь автосинхронизацию

Сделай **commit и push** в GitHub (например, создай test.py файл) — и через настроеную $автосинронизацию$ репозиторий автоматически обновится на сервере.

## (Дополнительно)
### 🧾 Добавь `.gitignore`

📌 В корне проекта должен быть `.gitignore`.

- Всё, что не должно попадать в GitHub (логи, кэши, временные файлы) — туда.
- Тогда в скриптах не нужно городить `git add . ':!что-то'` — Git сам будет игнорировать.
👉 Если `.gitignore` ещё нет — скопируй шаблон из другого репозитория и при необходимости добавь свои исключения.



**Когда .gitignore использовать не нужно**  
> [!note]- Подробнее
>
> 📌 Правильное решение — оставить `.md` в репозитории и управлять только через **`sparse-checkout`** на сервере.  
>
> 🔹 Тогда локально и на GitHub файлы будут жить и обновляться.  
> 🔹 А на сервере Git их просто не будет выгружать в рабочую директорию.  
>
> 🟡 `exclude` → файлы будут на сервере, но Git их не будет трекать.  
> 🟢 `sparse-checkout` → файлы вообще не попадут на сервер.

📂 Sparse-checkout настройки

Все правила, какие файлы и папки видны на сервере, а какие исключены, хранятся в специальной директории:  

```bash
/root/........./.git/info/
```

Чтобы посмотреть текущие правила, выполни команду:  

```bash
cat /root/........./.git/info/sparse-checkout
```

📌 Для `sparse-checkout` нужно:

Перейди в проект:
```bash
cd /root/........./
```

1. Включить режим sparse:
```bash
git sparse-checkout init --cone
```

2. Указать, что нужно исключить:
```bash
echo "...." >> .git/info/sparse-checkout
```

3. Применить:
```bash
git read-tree -mu HEAD
```
