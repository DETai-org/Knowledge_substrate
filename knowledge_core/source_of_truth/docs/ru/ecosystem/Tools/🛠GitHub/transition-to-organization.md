# Переход на организацию

## 1. Владелец ≠ человек.
Когда репозитории находятся в личном аккаунте, **владелец** — это человек. При переходе на организацию, владельцем становится сама **организация**. Это упрощает управление репозиториями и доступами.

- Легче добавлять и удалять людей без миграции прав.
- Меньше рисков из-за потери доступа (например, если кто-то из владельцев покидает проект).

## 2. Единственный вход для всех событий.
Переход на организацию позволяет настроить **единый вебхук** для всех репозиториев в организации. Это важная особенность, которая упрощает мониторинг и обработку событий.

- В организации можно настроить **Organization webhook** для всех репозиториев или выбрать нужные.
- Вместо использования множества вебхуков для каждого репозитория, можно использовать **GitHub App** для централизованного управления.

## 3. GitHub Projects для команды.
Проектные доски в GitHub теперь могут быть использованы для работы сразу с несколькими репозиториями. Это полезно для более гибкого управления задачами и проектами.

- Легче делиться досками на несколько репозиториев.
- Упрощается доступ для участников команды.

## 4. Секреты и ключи в одном месте.
Перевод секретов и переменных в **org-level** помогает централизовать управление доступами и безопасностью.

- Все ключи и токены для действий теперь можно хранить на уровне организации.
- Это улучшает безопасность, так как все данные централизованы, и доступ к ним проще контролировать.

## 5. Масштабирование.
Перевод на организацию упрощает добавление новых репозиториев, а также настройку работы с внешними сервисами, такими как **Uli Bot** для мониторинга и аналитики.

## 6. Стоимость.
Основные возможности организации — бесплатные. Платные функции, такие как **SSO**, **advanced security** и **code owners enforcement**, не критичны для старта.

## Что будет сложнее?
- **Перевыпуск токенов и вебхуков**: Нужно один раз настроить GitHub App или вебхук на уровне организации.
- **Перенос репозитория**: Репозитории можно легко перенести в организацию с сохранением всех данных и настроек.

## Как увязать это с ULI/ResourceTracker
В рамках организации создается **GitHub App** (например, **Uli Bot**), который позволяет интегрировать различные репозитории и отслеживать метрики, события и задачи.

(см. [github-app-bot](https://detai-org.github.io/Knowledge_substrate/ru/ecosystem/Tools/🛠GitHub/github-app-bot/)
