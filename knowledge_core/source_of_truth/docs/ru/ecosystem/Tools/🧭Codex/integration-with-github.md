
# Связь с GitHub

Codex и GitHub — это идеальная пара, потому что GitHub — место, где живёт код, а Codex — инструмент, который этот код пишет. И они умеют работать вместе **на нескольких уровнях**.

### Уровень 1: Codex внутри GitHub

**С февраля 2026 года Codex стал доступен прямо на GitHub** . Если у вас есть подписка **Copilot Pro+ или Copilot Enterprise**, вы можете запускать Codex, не выходя из интерфейса GitHub .

**Как это работает:**

- Заходите на **github.com** или в **GitHub Mobile** 
- Открываете репозиторий, issue или pull request
- Назначаете задачу Codex через Assignees dropdown — он автоматически начинает работу
- Codex анализирует задачу, пишет код и создаёт draft pull request для ревью 
- Можно оставлять комментарии с @codex — он учтёт правки и доработает

**Плюсы:**
- Не надо переключаться между инструментами
- Весь контекст (код, обсуждения, история) уже на месте
- Можно параллельно запускать несколько агентов для разных задач 

### Уровень 2: Codex в VS Code

Codex глубоко интегрирован в **Visual Studio Code** — основной редактор большинства разработчиков .

**Что доступно:**

- **Agent sessions view** — специальный раздел, где можно запускать и отслеживать работу Codex 
- Три типа сессий:
  - **Local** — быстрая интерактивная помощь
  - **Cloud** — автономные задачи, которые выполняются на GitHub
  - **Background** — асинхронные задачи (доступно для Copilot) 

**Важно:** VS Code должен быть версии **1.109 или новее** .

**Статистика:** У Codex **4.9 миллиона установок** в VS Code, выходит в среднем **1.8 релиза в день** — инструмент развивается бешеными темпами .

### Уровень 3: Codex Desktop App

Но Codex — это не только плагин. Это ещё и **отдельное десктопное приложение**, которое OpenAI выпустила сначала для macOS, а в марте 2026 года — и для Windows .

**Где скачать:**
- **Windows** — Microsoft Store (требуется Windows 10 версии 19041.0 или новее) 
- **macOS** — отдельное приложение от OpenAI 

**Что особенного в десктоп-версии:**

**1. Нативная песочница (sandbox)**
Для Windows OpenAI создала **первый в мире Windows-native агентный sandbox** . Это значит, что Codex может безопасно работать прямо в **PowerShell** и настоящих Windows-окружениях, без виртуальных машин и костылей .

**2. Параллельные агенты**
Можно запускать **несколько кодинг-агентов одновременно**, каждый в своём "worktree" — они не мешают друг другу и не создают конфликтов . Один пишет код, второй тестирует, третий ищет баги.

**3. Skills**
Это готовые сборки инструкций, скриптов и ресурсов, которые учат Codex работать с конкретными инструментами и процессами вашей команды . Один раз настроили — используете во всех проектах.

**4. Кросс-платформенность**
Сессии синхронизируются с вашим аккаунтом OpenAI: можно начать на Mac, продолжить на Windows — история и контекст сохраняются .

### Уровень 4: MCP (Model Context Protocol)

Для тех, кто хочёт полного контроля, есть **GitHub MCP server** — реализация Model Context Protocol, которая соединяет Codex напрямую с GitHub .

**Что можно автоматизировать:**
- Управлять issues (назначать, добавлять лейблы, отслеживать)
- Работать с пулл-реквестами
- Управлять доступом к репозиториям и командам
- Настраивать CI/CD (ветки, раннеры, секреты)
- Принимать приглашения в репозитории 

**Как подключить:**
Есть три способа — через Codex CLI, VS Code или десктоп-приложение. Везде настраивается через MCP Settings .

---

**Коротко:** Codex можно использовать **везде, где вы пишете код**.
- Любите веб-интерфейс? Работайте прямо на **GitHub.com**.
- Привыкли к редактору? Codex живёт в **VS Code**.
- Хотите мощь и контроль? Ставьте **Desktop App** для macOS или Windows.
- Нужна автоматизация? Подключайте через **MCP**.

И всё это синхронизируется между устройствами. Единая среда разработки, где бы вы ни находились.
