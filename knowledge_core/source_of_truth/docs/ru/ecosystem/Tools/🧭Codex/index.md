
# Codex

**Codex** — это AI-инструмент от компании OpenAI, который умеет **писать, анализировать и изменять код**. Если GPT работает с текстом и идеями, то Codex — это специалист именно по программированию.

## 🧩 Что это вообще такое

Представьте, что у вас есть напарник-программист, который:
- читает всю кодовую базу проекта
- понимает, как устроены файлы и папки
- знает, какие функции уже есть, а какие нужно дописать
- может объяснить, что делает любой кусок кода
- никогда не устаёт и работает мгновенно

Вот это и есть Codex.

## 🎯 Для чего он нужен

**Для создания приложений.** С Codex можно в буквальном смысле **наговорить приложение голосом**: описал задачу, показал скриншот интерфейса, объяснил логику — получил работающий код.

Но это не просто "генератор кода". Codex работает **в контексте всего проекта**:
- вы открываете папку с проектом
- Codex видит все файлы, все зависимости, всю структуру
- вы даёте задачу: "добавь страницу логина" или "перепиши этот компонент на TypeScript"
- Codex вносит изменения сразу во все нужные файлы

## 🔥 Чем это круто

**Для новичка** — вы можете создавать приложения, даже если только начали учиться программировать. Codex объяснит, что каждая строчка делает, поправит ошибки, покажет, как лучше.

**Для опытного разработчика** — Codex берёт на себя рутину: написание шаблонного кода, рефакторинг, тесты, документацию. Вы сосредотачиваетесь на архитектуре и сложных задачах.

## 🤝 Как Codex связан с GPT

Это два разных инструмента, которые отлично дополняют друг друга:

- **GPT** — архитектор, стратег, методолог. Он понимает задачу на высоком уровне, может разбить её на этапы, предложить архитектуру, объяснить, какие технологии использовать и почему. Подробнее про это в [[docs-as-executable-interface-for-agents|Документация как исполняемый интерфейс для агентов и людей]].

- **Codex** — исполнитель. Он берёт архитектуру, которую предложил GPT, и превращает её в работающий код. Причём сразу с учётом всех особенностей проекта, стиля кода, уже существующих решений.

## 🚀 Что уже сейчас можно делать

- **Писать код с нуля** — описали задачу, Codex написал файлы
- **Рефакторить** — "перепиши это на современный синтаксис"
- **Отлаживать** — "вот ошибка, найди, в чём дело, и исправь"
- **Писать тесты** — "сделай тесты для этой функции"
- **Документировать** — "напиши документацию по этому модулю"
- **Объяснять код** — "что здесь происходит?"

## 📍 Важно понимать

В нашей экосистеме Codex — один из ключевых инструментов для всего, что связано с разработкой: от небольших скриптов до сложных приложений.
