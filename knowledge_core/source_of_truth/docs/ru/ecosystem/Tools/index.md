# Tools

 *Инструменты экосистемы DET* 🧰 

💥💥💥 
Инструменты — это горизонтальный обеспечивающий слой экосистемы DET/DETai.

Если инструмент начинает требовать системной доработки — разработки интерфейсов, автоматизаций, API-интеграций, pipeline или собственной логики развития — он переходит в формат проекта **3 эшелона**.

В этом случае:

- инструмент становится объектом разработки,
    
- получает проектный контейнер внутри платформы DETai,
    
- но сохраняет свою горизонтальную роль в системе.
    

DET AI как технологический интерфейс экосистемы выполняет функцию усиления:  
он берёт инструмент, придаёт ему статус проекта, развивает его технологически — и тем самым усиливает всю экосистему.


> [!TIP] Это шаг к **автоматизированной среде разработки нового поколения**,  
где ты не «ручками таскаешь данные», а всё взаимодействует само по API.

### Инструменты:

| Категория                     | Инструменты и их подтипы           | Функция                                             | Взаимосвязь                                                                     |
| ----------------------------- | ---------------------------------- | --------------------------------------------------- | ------------------------------------------------------------------------------- |
| **Выполнение задач**          | **GitHub**                         | Репозитории, код                                    | GitHub → Codex (через PR), GitHub → Linear (связь задач и PR)                   |
| **Отслеживание прогресса**    | **GitHub → GitHub Projects**       | Управление проектами и задачами внутри орг или репо | GitHub Projects → GitHub (интеграция с репозиториями, задачи)                   |
| **Универсальные инструменты** | **GitHub → GitHub Bot**            | Автоматизация процессов, метки, управление PR       | GitHub Bot → GitHub (управление пулл-реквестами и метками)                      |
| **Отслеживание прогресса**    | **Linear**                         | Задачи, статусы, приоритеты                         | Linear → GitHub (связь с пулл-реквестами), Linear → Codex (отпр задачи в Codex) |
| **Отслеживание прогресса**    | **ClickUp**                        | Управление задачами и проектами для всех команд     | ?                                                                               |
| **Коммуникация**              | **Slack**<br>(Хотя ЕСЛИ есть ТГ ❓) | Уведомления, команды, синхронизация                 | Slack → GitHub (уведомления о событиях), Slack → Linear (уведомления о задачах) |

## Что это значит для DET

- ClickUp управляет идеями и задачами,
- GitHub хранит код,
- Codex анализирует и пишет код,
- Slack — интерфейс для общения с “умной средой”,
