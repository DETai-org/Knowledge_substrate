# Версии проектов в U.L.I.  
  
Большинство проектов размещаются в GitHub.  
  
Поэтому фиксация версии проекта обычно включает:  

`Git tag`  и `GitHub Release`

Создание версии происходит после завершения этапа разработки.  
  
Типичный цикл:  
```
Epic Issue → разработка → Git tag → GitHub Release
```
Про этот цикл подробнее смотри в [Work Model](https://detai-org.github.io/Knowledge_substrate/ru/ecosystem/DETai/U.L.I/3_Technical_Standards/work-model/work-model/)

## 🧩 Как работает в реальном проекте

Пример этапов:

|Этап|Что произошло|Версия|
|---|---|---|
|Завершена базовая архитектура сайта|Первый завершённый цикл|`0.1`|
|Стабилизация перед публичностью|Подготовка к выходу|`0.2`|
|Публичный запуск|Продукт объявлен стабильным|`1.0`|
___
## 🗂 Где указывать версию
  
### В GitHub Release  
  
там фиксируется техническое состояние системы. Подробнее см.  [Release Fixation Standard](https://detai-org.github.io/Knowledge_substrate/ru/ecosystem/DETai/U.L.I/3_Technical_Standards/release-fixation-standard/)
  
### В README проекта  
  
пример:  
```
♻️ Версия: v0.1  
⏯️ Статус: released  
📅 Дата открытия: 2026-02-03  
📅 Дата закрытия: 2026-03-01
```
  
README отражает **смысловой статус проекта**, а Release фиксирует **техническое состояние системы**.

# Практический принцип  
  
В U.L.I. версии используются как инструмент управления развитием проектов и знаний.  
  
Версия означает:  
  
- завершение этапа разработки  
- признание нового устойчивого состояния системы  
- синхронизацию команды вокруг текущего состояния проекта


___
## 🧮 Как повышать версию

Версия повышается при завершении нового логического этапа.

- `0.1 → 0.2` — продукт находится в стадии формирования при этом и тут могут быть  циклы развития
- `0.x → 1.0` — продукт признан стабильным и готовым к публичной ответственности  — это стадия, на которой проект *уже выполняет свою ключевую функцию* и *начинает приносить тот ресурс*, ради которого создавался.
- Важно: версия **не обязана доходить до 0.9** перед переходом на 1.0.

Повышение версии — это решение уровня зрелости, а не просто увеличение цифры.
Оно фиксирует не просто улучшение проекта, а реализацию **стратегии развития**, связанной с текущей эпохой (Milestones 📅).

Новая версия означает, прирост объёма того ресурса, который ожидался от появления этого конкретного проекта
