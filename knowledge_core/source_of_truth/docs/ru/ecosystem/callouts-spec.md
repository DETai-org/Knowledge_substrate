# Спецификация callout-блоков (Obsidian → Material for MkDocs)

Этот документ фиксирует единый стандарт преобразования callout-синтаксиса Obsidian в admonition-блоки Material for MkDocs.

## Цель

Обеспечить единообразное отображение информационных блоков в документации и устранить разночтения по типам/цветам.

## Каноническое соответствие типов

- `INFO -> info`
- `WARNING -> warning`
- `TIP -> tip`
- `NOTE -> note`
- `QUOTE -> quote`

## Правило обработки

1. В markdown-файлах допускается исходный Obsidian-синтаксис:

   ```md
   > [!TIP] Текст подсказки
   ```

2. На этапе сборки MkDocs этот синтаксис автоматически преобразуется в admonition-формат Material for MkDocs:

   ```md
   !!! tip "Текст подсказки"
   ```

3. Для сохранения предсказуемого поведения отключается авто-конвертация алиасов (например, `IMPORTANT`, `HINT`), и используются только явно зафиксированные типы из таблицы выше.

## Примечание по совместимости

Для корректного рендера преобразованных блоков в проекте должны быть включены markdown-расширения:

- `admonition`
- `pymdownx.details`
