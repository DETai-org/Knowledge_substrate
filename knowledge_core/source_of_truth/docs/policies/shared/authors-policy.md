# Authors Policy

Технический JSON-источник: [authors.json](../../schemas/authors.json)

## Правило

- `authors.json` — общий справочник авторов для сайтов.
- Профессиональный сайт DETai использует этот справочник для отображения автора, роли, аватара и будущих дополнительных характеристик.
- Персональный сайт не показывает author-фильтр в блоге, потому что там один автор. Данные автора могут оставаться в frontmatter как служебная совместимость, но не должны создавать отдельную навигацию.
- Новые авторы для профессионального сайта добавляются в `authors.json`, а затем могут использоваться в `administrative.authors` постов.

## Поля автора

| Field | Meaning |
| --- | --- |
| `id` | Стабильный ID автора. |
| `display_name` | Отображаемое имя, которое должно совпадать с именем в `administrative.authors`. |
| `slug` | Стабильный slug автора. |
| `type` | Тип сущности, обычно `person`. |
| `role` | Роль автора на сайте. |
| `avatar` | Путь к аватару для UI. |

## Описательная копия

Эта политика описывает [authors.json](../../schemas/authors.json), но не является источником данных. Если меняется имя, роль, slug, avatar или набор авторов, менять нужно JSON, а эту таблицу обновлять как копию.
