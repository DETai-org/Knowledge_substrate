# Categories Policy

Технический JSON-источник: [categories.json](../../schemas/post_documents/categories.json)

## Правило

- У поста может быть одна или несколько категорий.
- Категории описывают поперечные темы и фильтры.
- Категория может быть общей или уникальной для одного сайта.
- Emoji категории является частью визуального представления в интерфейсе.
- `landing_page: true` означает, что категория имеет утверждённую посадочную страницу/landing-назначение в таксономии. Категории без этого флага могут использоваться как фильтры и метки, но не считаются утверждёнными category landing pages.

## Shared vs unique

- `shared` — категория доступна обоим сайтам
- `detai-only` — категория только для профессионального сайта
- `personal-only` — категория только для персонального сайта

## Уникальные категории

### DETai-only

- `category:branding-det`

### Personal-only

- `category:personal-lesgaft-study-notes`
- `category:personal-health-practices`
- `category:personal-rhyme-poetry`

## Удалённая категория

- `category:about-me` больше не используется и не должна назначаться постам.
