# Цитаты и атрибуция в блог-постах

Данная политика описывает единые правила оформления цитат и указания источников в блог-постах.  
Её цель — обеспечить корректный рендер цитат на сайте, единый визуальный стиль и устойчивую автоматическую обработку контента.

---

## Область действия

Политика применяется ко всем блог-постам, публикуемым через Markdown и рендерящимся с использованием remark/rehype и Tailwind Typography (`prose`) в проектах:

- `detai-site`
- `personal-site`

---

## Как цитаты появляются на сайте (важно)

Автор текста **не работает напрямую с Markdown-разметкой** и **не обязан знать технические детали рендера**.

Фактический процесс выглядит так:

1. **Автор (человек)** пишет текст в обычной форме:
   - выделяет цитаты кавычками (`«…»` или `“…”`);
   - указывает автора и источник привычным литературным способом.

2. **Агент / пайплайн подготовки контента**:
   - преобразует текст в Markdown;
   - помечает цитаты специальной разметкой;
   - сохраняет результат в базе данных / репозитории.

3. **Сайт**:
   - рендерит Markdown в HTML;
   - применяет единые стили и правила обработки цитат.

Все технические механизмы описаны ниже **для понимания принципов**, а не как инструкция для автора.

---

## Термины

### Блок цитаты (`blockquote`)

`blockquote` — это **технический формат блока цитаты**, используемый внутри Markdown и HTML.

Он:
- появляется автоматически на этапе подготовки контента;
- **не пишется вручную автором**;
- используется системой для того, чтобы однозначно отличать цитаты от обычного текста.

Технически в Markdown он выглядит так:

```markdown
> Текст цитаты
````

Символ `>` — это **внутренний маркер**, а не требование к автору.

## Каноническая структура цитаты

### 1) Текст цитаты

Одна цитата может содержать:

* один или несколько предложений;
* один или несколько абзацев;
* при этом оставаться **одной логической цитатой**.

Для автора это всегда просто цитируемый фрагмент текста.

---

### 2) Атрибуция (обязательная подпись)

Если у цитаты есть источник, **атрибуция обязательна**.

В Markdown-представлении атрибуция оформляется:

* как **отдельный абзац**;
* **последней строкой внутри блока цитаты**;
* **строго в круглых скобках**.

Канонический паттерн:

```markdown
> Текст цитаты…
>
> (Автор, Источник / детали)
```

---

## Правила заполнения атрибуции

### 1) Автор — до первой запятой

Первый сегмент текста **до первой запятой** трактуется системой как имя автора.

Правильно:

```markdown
(Айн Рэнд, «Атлант расправил плечи», ч. 3, с. 186)
```

Неправильно:

```markdown
(А., Рэнд, «Атлант расправил плечи»)
```

Если используются инициалы, **запятая внутри имени не ставится**.
Запятая ставится **после полного указания автора**.

Допустимо:

```markdown
(А. Рэнд, «Атлант расправил плечи»)
```

---

### 2) Источник — всё после первой запятой

Всё, что следует после первой запятой:

* считается источником;
* может содержать дополнительные запятые;
* не требует специального форматирования.

Примеры допустимых источников:

* название произведения;
* часть, глава, страница;
* сборник, перевод, год.

---

### 3) Если источник неизвестен

Если известен только автор, допустима атрибуция без источника:

```markdown
(М. Монтень)
```

---

## Кавычки и «умные кавычки»

### 1) Рекомендуемое поведение автора

Автору **рекомендуется**:

* оформлять прямые цитаты кавычками (`«…»` или `“…”`).

Это повышает читаемость текста и сохраняет авторский стиль.

---

### 2) Автоматическая подстраховка системы

Если в цитате **отсутствуют кавычки**, система:

* автоматически добавляет декоративные кавычки при рендере.

Если кавычки **уже присутствуют**, система:

* не добавляет дополнительные кавычки;
* предотвращает визуальное дублирование.

---

## Поведение рендера (гарантии)

При соблюдении канонического паттерна система гарантирует:

1. **Разделение цитаты и подписи**

   * подпись не попадает внутрь кавычек;
   * подпись визуально отделена от текста цитаты.

2. **Визуальное оформление подписи**

   * имя автора выводится отдельной строкой;
   * источник — второй строкой;
   * меньший размер шрифта;
   * спокойный (muted) цвет;
   * выравнивание по правому краю;
   * отсутствие курсива.

3. **Устойчивость к вариациям текста**

   * несколько абзацев внутри цитаты;
   * отсутствие кавычек у автора;
   * сложные источники с несколькими запятыми.

## Примеры (канон)

### Пример A — цитата с источником

```markdown
> «Если бы я предался сожалениям о прошлом, это было бы изменой истине».
>
> (Айн Рэнд, «Атлант расправил плечи», ч. 3, с. 186)
```

### Пример B — цитата без кавычек (кавычки добавит система)

```markdown
> Если бы я предался сожалениям о прошлом, это было бы изменой истине.
>
> (Айн Рэнд, «Атлант расправил плечи», ч. 3, с. 186)
```

### Пример C — только автор

```markdown
> «Познай самого себя».
>
> (Сократ)
```

## Анти-примеры (не допускается)

### Атрибуция в той же строке

```markdown
> «Текст цитаты…» (Автор, Источник)
```

### Атрибуция без круглых скобок

```markdown
> «Текст цитаты…»
>
> — Автор
```
