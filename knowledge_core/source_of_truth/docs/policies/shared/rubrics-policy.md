# Rubrics Policy

Технический JSON-источник: [rubrics.json](../../schemas/post_documents/rubrics.json)

## Правило

- У каждого поста ровно одна рубрика.
- Рубрика задаёт главный смысловой вектор поста.
- Если пост не попадает в основные оси персонального сайта, используется рубрика `rubric:det-notes` с персональным label override `Заметки`.

## ID и route slugs

Рубрики отличаются от категорий тем, что у них есть локализованные `routeSlugs`: разные URL для разных языков и иногда разные URL-label на разных сайтах. При этом все эти URL опираются на один стабильный ID рубрики, например `rubric:det-notes`. Именно ID используется в frontmatter и данных постов, а `routeSlugs` отвечают за публичные адреса вида `/{lang}/blog/{slug}`.

## Актуальная структура

### Shared

- `rubric:shadow-and-light`
- `rubric:personality-types-and-existential-strategies`
- `rubric:orientation-toward-overcoming`
- `rubric:det-notes`

### DETai-only

- `rubric:self-promotion-skills`

### Personal-only

- `rubric:personal-key-milestones`
- `rubric:about-me`

## Дополнительно

- На персональном сайте label для `rubric:det-notes` отображается как `Заметки`.
- На профессиональном сайте label для той же рубрики отображается как `Заметки DET`.
- Рубрики имеют localized `routeSlugs`; страницы рубрик генерируются маршрутом блога `/{lang}/blog/{slug}`.
