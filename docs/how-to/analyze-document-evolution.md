# How-to: анализ эволюции документа по версиям

Этот how-to описывает пошаговый процесс, как анализировать эволюцию любого документа в репозитории на основе индекса версий и Git-истории.

## Источник версий

Используй индекс версий:

- [knowledge_core/source_of_truth/docs/ecosystem/version_index.yaml](../../knowledge_core/source_of_truth/docs/ecosystem/version_index.yaml)

> В индексе хранятся: `id`, `title`, список `versions`, `version`, `commit`, `date_ymd`, `path`.

---

## Шаги анализа

### Шаг 1. Определи документ

1. Открой `version_index.yaml`.
2. Найди запись с нужным `id` (например, `constitution`).
3. Зафиксируй:
   - `title` документа,
   - список версий,
   - список commit-хэшей,
   - путь к файлу (`path`).

### Шаг 2. Ограничь период анализа (если нужно)

Если запрос содержит период (например, «за 2026»):

1. Отбери из индекса только версии с нужным `date_ymd`.
2. Если нужны все изменения внутри периода, дополнительно проверь Git-лог по файлу `path` за указанный диапазон дат.

### Шаг 3. Построй сравнения между версиями

Для смыслового анализа сравнивай версии попарно:

- соседние версии (`v1 -> v2`, `v2 -> v3`),
- или конкретные версии из запроса (`v2 -> v5`).

Базовая логика:

1. Возьми commit старой версии.
2. Возьми commit новой версии.
3. Построй diff по `path` между этими commit.
4. Выдели изменения по смысловым блокам (разделы, принципы, правила, роли, ограничения).

### Шаг 4. Сделай смысловую классификацию изменений

Для каждого diff зафиксируй:

- **что изменилось** (формулировки, структура, полномочия, правила);
- **где изменилось** (раздел/подраздел документа);
- **влияние** (прояснение, ужесточение, расширение, переопределение).

### Шаг 5. Сформируй ответ под тип запроса

#### Запрос: «проанализируй эволюцию constitution за 2026»

Выдай:

1. хронологию версий за 2026;
2. что менялось между каждой парой версий;
3. общий тренд эволюции документа за год.

#### Запрос: «дай смысловые изменения между v2 и v5»

Выдай:

1. прямое сравнение `v2` и `v5`;
2. ключевые отличия по разделам;
3. итог: какие нормы/правила изменились принципиально.

#### Запрос: «покажи, когда поменялись правила делегирования»

Выдай:

1. список версий/коммитов, где затронуты разделы про делегирование;
2. краткую хронологию изменений именно по делегированию;
3. указание, в какой версии правило стало текущим.

---

## Рекомендуемый промпт для агента

```text
Проанализируй эволюцию документа с id=<ID> на основе файла
knowledge_core/source_of_truth/docs/ecosystem/version_index.yaml.

Сделай:
1) список релевантных версий и commit;
2) смысловые diff между нужными версиями;
3) выводы по изменениям правил.

Если указан период, учитывай только его.
Если указан аспект (например, делегирование), фокусируйся только на релевантных разделах.
```

---

## Правило точности

При ответах всегда указывай:

- `id` документа,
- `title` документа,
- `version`,
- `commit`,
- дату (`date_ymd`),
- путь (`path`).

Это обеспечивает воспроизводимость анализа и проверяемость выводов.
